## Deploy blockers to verify before Render

1. **Run `npm run build` locally.** `next.config.js` has no `eslint.ignoreDuringBuilds` / `typescript.ignoreBuildErrors`, so Next builds ALL ~38 routes under `app/` (not just the 19 in nav) and any TS/ESLint error fails the deploy. Paste errors and I'll fix.
2. **`package-lock.json`** — present and must be in sync (`render.yaml` runs `npm ci`, which hard-fails on a stale lockfile). If you changed deps, run `npm install` and commit the lockfile.
3. **`NODE_OPTIONS=--max-old-space-size=768`** in `render.yaml` is tight for a build with puppeteer + tesseract + this many routes. If the build OOMs on Render, bump it.
4. **`CHANGELOG.md`** and **`app/api/econ-calendar/events.json`** must be committed (not gitignored) — both are read from disk at runtime.
